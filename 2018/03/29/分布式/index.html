分布式

什么是分布式

本身单个节点分部成多台主机进行处理

1、任务分解

2、节点通信

分布式和集群的关系

分布式是一个业务拆分成多个子业务，部署在不同的服务器上 （解决高可用问题）

集群：同一个业务部署在多个服务器上（解决高并发问题，负载均衡）

电商平台：用户、商品、订单、交易

缓存、限流、降级

###session跨域共享问题

1、session sticky

2、session replication

3、session集中存储(db,缓存服务器，redis)

4、cookie(主流)

网络通信

网络通信是分布式的基础

网络协议：TCP/IP和UDP/IP

IO:BIO,NIO,AIO

TCP 5层：应用层，传输层，网络层，链路层，物理传输层

OSI模型：应用层，传输 层，网络层，链路层，物理传输层 ，表达层，会话层

3次握手协议、4次挥手协议（TCP是全双工的）

Multicast（组播）

单播

点对点通讯，每次只有2个实体相互通信

广播

一对多通讯

组播

序列化

把对象转化为字节序列的过程称之为对象的序列化反之称之为反序列化

java本身序列化存在的问题

1、序列化数据结果比较大、传输效率比较低

2、不能跨语言对接

XML编码格式的对象序列化机制成为了主流

恰当的序列化协议不仅可以提高系统的通用性、强壮性、安全性、优化性能，同时还能让系统更加容易调试和扩展。

实现序列化步骤 1、集成Serializable，2、生成一个seriaVersionUID

seriaVersionUID的作用

能够保证反序列化对象和反序列化以后的对象是同一个

序列化并不报错静态变量的状态

transient不参与序列化

通讯协议

###URI和URL

URI:web服务器资源的名字， index.html

http://www.baidu.com:80/java/index.html

schema:http/https/ftp

host:web服务器的ip地址或者域名

port:服务端端口

path:资源访问路径

query-string:查询参数

状态码

htpps 1.1 版本协议定义了五种类型的状态码

1xx 提示信息

2xx 成功

3xxx重定向

4xx客户端错误

5xx服务端错误

http协议的特点

1、无状态：每次请求都是独立的，和下次请求没有关联关系 (cookie +session)

2、多次请求

3、基于TCP协议

HTTPS

基于SSL/TLS

HTTPS的工作原理

1、使用对称加解密

2、密钥是公开的，所有的客户端都可以拿到密钥

3、针对不同的客户端使用不同的密钥

协商过程是没有加密的，所以还会出现被截断的问题

4、使用非对称加密

问题：客户端如何拿到公钥

服务器端把公钥发给每一个客户端

服务器端把公钥放在远程服务器，客户端可以请求到

让浏览器保存所有公钥

5、公钥被掉包的问题永远存在。

6、使用第三方机构来解决。

通过第三方机构，使用第三方机构的私钥对我们的 需要传输的公钥进行加密

7、数字证书包含：公司信息、网站信息，数据公钥

RESTful的基本概念

1、在REST中，一切的内容都被认为是一种资源

2、每个资源都由URI唯一标识

3、使用同一的接口处理资源请求

4、无状态

分布式通信框架-RMI

什么是RPC， Remote procedure call protocal

RPC协议是一个规范。dubbo 、Thrif、RMI、Webservice、Hessain

网络协议和网络IO对于调用端和服务端来说是透明。
