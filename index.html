分布式是一个业务拆分成多个子业务，部署在不同的服务器上 （解决高可用问题）

集群：同一个业务部署在多个服务器上（解决高并发问题，负载均衡）

序列化

把对象转化为字节序列的过程称之为对象的序列化反之称之为反序列化

java本身序列化存在的问题

1、序列化数据结果比较大、传输效率比较低

2、不能跨语言对接

XML编码格式的对象序列化机制成为了主流

恰当的序列化协议不仅可以提高系统的通用性、强壮性、安全性、优化性能，同时还能让系统更加容易调试和扩展。

实现序列化步骤 1、集成Serializable，2、生成一个seriaVersionUID

seriaVersionUID的作用

能够保证反序列化对象和反序列化以后的对象是同一个

序列化并不报错静态变量的状态

transient不参与序列化

kettle

kettle是一个独立的产品，但它包括了在ETL开发和部署阶段用到的多个程序，有独立的功能，也或多或少的依赖其他程序。

###主要功能：

1、spoon :集成开发环境，提供了一个图形用户界面，用于创建、编辑作业或也可以用于执行/调试作业或转换，它也有性能监控的功能。

2、kitchen:作业的命令运行程序，可以通过shell脚本来调用

3、pan：转换的命令运行程序，和kitchen一样通过shell脚本调用。执行作业

4、Carte：轻量级HTTP服务器（基于jetty），后台运行，监听HTTP请求来运行一个作业。用于kettle集群

下载地址

https://sourceforge.net/projects/pentaho/files/

###文件格式：

归档文件的命名格式按照pdi-ce-version.extension格式，

pid代表Pentaho Data Integration

ce 代表了Community Edition

子系统1~3 ：数据剖析，增量数据捕获 抽取

子系统1：数据剖析系统

这个子系统的目标是分析不同数据源的结构和内容。

数据剖析提供了类似行统计NULL值个数统计等简单的统计项。

该子系统的目标是捕获源系统里数据的变化。

当前kettl里没有特别的工具可以完成这一功能，但kettle里的一些步骤可以通过时间戳，快照方式获得变化的数据。

子系统3：抽取系统

该子系统是要从不同的数据源抽取数据，并输入到ETL流程里。

kimball明确区分了基于文件的和基于流的两种抽取。

kettle认为抽取的主要区别不是文件或流，而是静态或者动态。

子系统6：审计维度

审计维度表示一类特殊的维度表，数据仓库里的所有事实都和设计维度表关联，设计维度表包含了对事实表变更的元数据，如加载数据的日期和时间、数据的质量指标等。

centos安装jdk

安装之前先检查一下系统有没有自带open-jdk

命令：

rpm -qa |grep java

rpm -qa |grep jdk

rpm -qa |grep gcj

如果没有输入信息表示没有安装。

如果安装可以使用rpm -qa | grep java | xargs rpm -e –nodeps 批量卸载所有带有Java的文件 这句命令的关键字是java

首先检索包含java的列表

yum list java
检索1.8的列表
yum list java-1.8
安装1.8.0的所有文件
yum install java-1.8.0-openjdk* -y
使用命令检查是否安装成功
java -version
到此安装结束了。这样安装有一个好处就是不需要对path进行设置，自动就设置好了

javap命令

javap定义

javap是java文件的分解器，即是jdk自带的反汇编器，可以查看java编译器为我们生成的字节码。通过它，我们可以对照源代码和字节码，从而了解很多编译器内部的工作。

语法：

javap [命令选项] class ….

Javap 命令用于解析类文件，其输出取决于所用的选项。若没有使用选项，javap将输出传递给他的类public 域及方法，javap将其输出到标准输出设备上。

命令选项：

-help 输出javap的帮助信息

-l 输出行及局部变量表

-b 确保与jdk1.1 javap的向后兼容性

-public 只显示public类及成员

-protected只显示protected和public 类以及成员

-package 只显示包、public和protected类以及成员

-private 显示所有类和成员

-J[flag] 直接将flag传给运行时系统

-s输出内部类型签名

-c输出类中各个方法未解析的代码，即构成java字节码的指令

-verbose 输出堆栈大小、各方法的locals以及args数，和class文件的编译版本

-classpath[路径] 指定javap用来查找类 的路径。如果设置了该选项，则他将覆盖缺省值或CLASSPATH环境变量，目录用分号冒号分割

nohup

在处理命令前加上nohup即可，标准输出和标准错误缺省会被重定向到nohup.out 文件中，一般我们可以在结尾加上& 来讲命令同时放入到后台运行，也可用 “?filename 2>&1”来更改缺省的重定向文件名。

&

加在一个命令的最后可以把这个命令放到后台执行，如gftp &

ctrl +z

可以将一个正在前台执行的命令放到后台，并且处于暂停状态，不可执行

####登录到msyql

5.7版本默认对于root帐号有一个随机密码，可以通过 grep “password” /var/log/mysqld.log获得，root@localhost: 此处为随机密码
运行mysql -uroot -p 回车
粘贴随机密码

操作

默认的随机密码是没办法直接对数据库做操作的，需要修改密码，然后，5.7版本用了validate_password密码加强插件，因此在修改密码的时候绝对不是 123456 能糊弄过去的。需要严格按照规范去设置密码
但是，如果想让密码简单点也可以，降低安全策略， 登录到mysql客户端执行如下两条命令

set global validate_password_length=1;

set global validate_password_policy=0;

这样就能设置简单的密码了，但是密码长度必须是大于等于4位

这条命令是解出all.tar包中所有的文件
